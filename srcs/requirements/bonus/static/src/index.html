Project Overview

Inception is a system administration project focused on mastering Docker by building a small infrastructure from scratch. The goal is to virtualize multiple services using Docker Compose inside a virtual machine, ensuring best practices in containerization and deployment.

Mandatory Requirements

  • Each service runs in its own container.
  • Containers must be built from Alpine or Debian.
  • No pre-built images allowed (except Alpine/Debian).

Services:

  • NGINX: TLSv1.2 or TLSv1.3 only, entry point via port 443.
  • WordPress: with php-fpm (no NGINX inside).
  • MariaDB: database only (no NGINX inside).

Infrastructure Rules:

  • Volumes for WordPress database + website files.
  • Custom Docker network connecting services.
  • Containers auto-restart on crash.
  • Domain: <login>.42.fr mapped to VM IP.
  • Secure credentials with .env
  • At least 2 DB users, including an admin (username cannot contain admin).

Bonus Part

  • Redis cache for WordPress.
  • FTP server linked to WordPress volume.
  • Static website (Inception project Overview).
  • Adminer for DB management.
  • cadvisor for monitoring.
